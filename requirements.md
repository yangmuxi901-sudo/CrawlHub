# A 股投资者关系活动记录表 PDF 下载器 - 需求规格说明书

**版本：** 1.0
**日期：** 2025-02-20
**状态：** MVP

---

## 1. 项目概述

### 1.1 项目目标

开发一个独立的 Python 命令行工具，自动化下载 A 股上市公司的《投资者关系活动记录表》PDF 文件。

### 1.2 范围

- ✅ 独立运行，不依赖任何现有系统
- ✅ 命令行交互
- ✅ 增量下载（高水位线机制）
- ✅ 本地存储 PDF + SQLite 状态记录

### 1.3 不在范围

- ❌ 与任何 ERP/CRM/OA 系统集成
- ❌ Web 界面 / GUI
- ❌ 数据解析（PDF 内容提取）
- ❌ 定时调度（手动运行）

---

```sql
CREATE TABLE download_history (
```

#### 3.3.2 逻辑规则

| 规则 ID | 描述 |
|---------|------|

### 3.4 API 数据获取

#### 3.4.1 数据源优先级

```
1. 巨潮资讯网底层 JSON API (手写 requests)  [优先]
```

#### 3.4.2 强制要求

| 要求 ID | 描述 |
|---------|------|
| API-01 | 禁止解析任何 HTML 页面 |
| API-02 | 必须直接请求 JSON 格式的 Ajax/API 接口 |
| API-03 | 只保留标题含"投资者关系"或"投资者调研"的记录 |

#### 3.4.3 巨潮资讯 API 参数


### 3.5 PDF 下载

#### 3.5.1 文件命名规范

```
data/ir_pdfs/{ticker}_{publish_date}_{title}.pdf
```

#### 3.5.2 标题清洗规则

必须移除以下非法字符：`\ / : * ? " < > |`

```python
# 示例
原标题 = "2024 年投资者调研/活动记录表:问答*纪要"
清洗后 = "2024 年投资者调研_活动记录表_问答_纪要"
```

#### 3.5.3 下载要求

| 要求 ID | 描述 |
|---------|------|
| DL-01 | 设置 `timeout=15` |
| DL-02 | 必须使用 `try-except` 包裹 |
| DL-03 | 下载失败不中断，继续下一个 |
| DL-04 | 文件已存在则跳过 |

### 3.6 防御性编程

| 机制 | 要求 |
|------|------|
| 反爬休眠 | `time.sleep(random.uniform(2, 5))` |
| 网络超时 | `timeout=15` |
| 异常处理 | 单文件失败不中断整体流程 |
| 日志输出 | 关键节点打印中文状态 |

## 4. 用例 (Use Cases)

### UC-01: 首次运行 - 全量下载

**前置条件：**
- 公司列表.csv 存在
- 数据库为空

**主流程：**
1. 用户运行脚本
2. 脚本创建 `data/ir_pdfs/` 和 `data/sync_record.db`
3. 遍历公司列表
4. 每只股票从 2024-01-01 开始抓取
5. 过滤出"投资者关系"记录
6. 下载 PDF 到本地
7. 更新数据库

**预期结果：**
- 所有新 PDF 下载成功
- 数据库记录每只股票的最新日期

---

### UC-02: 增量运行 - 只下载新记录

**前置条件：**
- 数据库已有上次运行记录

**主流程：**
1. 用户再次运行脚本
2. 脚本查询每只股票的 `last_publish_date`
3. 只抓取该日期之后的新记录

**预期结果：**
- 已下载的 PDF 不重复下载
- 只下载新增的 PDF

---

### UC-03: 网络异常 - 单文件失败

**前置条件：**
- 某 PDF 链接失效或超时

**流程：**
1. 下载某 PDF 时返回 404 或超时
2. 脚本捕获异常，打印 Error 日志
3. 继续下载下一个文件

**预期结果：**
- 程序不崩溃
- 其他文件正常下载

---

### UC-04: 文件已存在 - 跳过

**前置条件：**
- 某 PDF 文件已存在于本地

**流程：**
1. 检查文件路径
2. 文件已存在

**预期结果：**
- 跳过下载
- 计数 +1

---

### UC-05: 无效股票代码 - 跳过

**前置条件：**
- 公司列表中存在格式错误的 ticker

**流程：**
1. 解析 ticker 格式
2. 发现不是 `交易所。代码` 格式

**预期结果：**
- 打印跳过日志
- 继续处理下一只股票

## 7. 技术实现细节

### 7.1 巨潮资讯 API 参数格式（已验证）

```
URL: http://www.cninfo.com.cn/new/hisAnnouncement/query
Method: POST
Content-Type: application/x-www-form-urlencoded; charset=UTF-8

参数:
- pageNum: 页码
- pageSize: 每页数量 (最大 100)
- column: 市场类型 (szse/sse/bse)
- tabName: 'fulltext'
- stock: '' (全局搜索时为空)
- searchkey: 搜索关键词
- seDate: '开始日期~结束日期' (例如：'2024-01-01~2025-02-20')
- sortName: 'announcementTime'
- sortType: 'desc'
- isHLtitle: 'true' (返回带高亮标记的标题)

返回数据示例:
{
  "announcements": [
    {
      "secCode": "300054",
      "announcementTitle": "<em>投资者关系</em>活动记录表",
      "announcementTime": 1739980800000,
      "adjunctUrl": "finalpage/2025-02-20/1222594208.PDF"
    }
  ],
  "totalAnnouncement": 100
}
```

**注意事项:**
- 必须先用 `requests.Session()` 获取首页 Cookie
- 特定股票查询 API 返回 0 数据，需采用全局搜索然后过滤的策略
- 标题中包含 `<em>` 高亮标签，需要用正则清理：`re.sub(r'<[^>]+>', '', title)`
- PDF 下载地址：`http://static.cninfo.com.cn/{adjunctUrl}`

### 7.2 批量获取策略

由于特定股票查询 API 无法正常返回数据，采用批量获取策略：

1. 计算所有股票的最早高水位线日期
2. 按市场（szse/sse/bse）批量获取该日期至今的所有投资者关系记录
3. 构建 `{stock_code: [records]}` 映射
4. 遍历公司列表，从映射中获取对应股票的记录并下载

**优点：**
- 避免逐只股票查询的低效
- 只需 3 次 API 调用（每市场一次）
- 适合大批量股票下载

**缺点：**
- 首次运行时间较长（需处理大量数据）
- 无法针对单只股票单独更新
