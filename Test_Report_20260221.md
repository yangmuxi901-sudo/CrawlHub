## 4. 风险评估报告

1. **已发现的高级风险（逻辑层）：** 
   - 增量逻辑的跳出缺陷（BUG-01）会导致：一旦有一个文件因各种原因残留在本地，但数据库中的记录较旧，整个增量脚本的效能会大幅降低，每次都要重新请求和比对已有的几百条旧记录，白白浪费 API 额度和执行时间。
2. **已发现的业务风险（Web展示层）：**
   - 进度的硬编码（BUG-02、BUG-03）会导致若更换不同规模的 公司列表.csv，前端 Web 任务卡片的进度条（Progress bar）出现逻辑错误或始终为 0% 状态，极大地影响了产品的可用性体验。
3. **缓解建议：**
   - 在 web/api.py 里，实现统一的配置读取，或允许脚本将“公司总数”写入日志头部供正则读取。
   - 巨潮网 API 的网络容错目前仅在抓取页发生，未在“单文件下载 download_pdf”中针对 403 / 502 等加入重试机制（目前发生异常仅返回 False），在长时跑批（如总共 495 家公司）时容易产生数据漏损。

## 5. 测试结论

系统目前已达到 **MVP (最小可行性产品) 的可用状态**，核心的批量下载、Web 管理调度、数据库状态追踪逻辑清晰且初步跑通。

**不建议立即上线（生产使用）**。需优先修复 BUG-01（严重影响增量运行效率和数据一致性）和 BUG-02/BUG-03（直接导致前端进度卡死）。
修复完上述缺陷后，该项目具备在实际生产环境中稳定抓取 A股投资者关系记录的能力。
